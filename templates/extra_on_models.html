Dataset provided by Santander had dimension of (200000,202).It was a binary dataset where we have predict will customer make the Transaction (0 for customer will not make the transcation, 1 for transcation will happen).Evaluation metric was area under the ROC curve between the predicted probability and the observed target. For the metrics in the dataset no specfic information was provided columns were named as VAR_1, VAR_2 so on.

Standard scalar was applied which improved the model performace by 0.2 (AUC of ROC curve) irrespective of model used. EDA was done using libraries like Matplotlib, seaborn.

For the modeling diffrent algorithems were consider like Naive Bayes, Lightgbm, XGboost, Logestic Regression, Random Forest

Area under ROC curve for Naive Bayes is - 0.88762

Link to Naive Bayes Notebook: Naive Bayes on Santander Customer Transactions

Area under ROC curve for Lightgbm is - 0.89076

Link to Lightgbm Notebook: Lightgbm on Santander Customer Transactions

Area under ROC curve for Lightgbm and Naive bayes with Logestic Regression is - 0.89659

Link to Lightgbm + Naive Bayes + logestic Notebook: Logestic on Santander Customer Transactions

Random Forest and XGboost were not giving the good results as compared to the Lightgbm.

This model was deployed using FASTAPI.

The objective to build a model, learned using historical data, that will determine an employee's access needs, such that manual access transactions (grants and revokes) are minimized as the employee's attributes change over time. The model will take an employee's role information and a resource code and will return whether or not access should be granted.

Various feature engineering techniques were used on the data. Like frequency encoding, One Hot encoding. Results with both the techniques were encouraging. Smote oversampling method was also used.

Various M.L algorithems were tried like SVM, Logestic Regression, Naive Bayes, Random Forest, XGboost, Lightgbm. XGboost gave bests result.

Link to EDA: EDA of Aamzon resource access

Link to XGboost: XGboost on Aamzon resource access

The model was able to predict to allow access to employee when actually needed by 91.29%. Saving time and resources of the company.

The model was able to predict not to allow access to employee when actually not needed by 67.31%. Saving time and resources of the company.

This diffrence between the prediction accuracy of diffrerent classes is due to the imbalance in the dataset.
